## 主要技術里程碑

### §J.3.e.X Path β — Native RIFE Vulkan FRUC

完整 RIFE-v4.25-lite Vulkan compute pipeline：
- `.param` parser + fp16 weight loader（§J.3.e.X Phase 1+2）
- 9 個 compute shaders（Conv2D / Activation / BinaryOp / Concat / Crop / Interp / PixelShuffle / Eltwise / Deconvolution / rife.Warp）
- 389-layer graph executor with VkPipelineCache cross-launch persistence
- flow extraction（tensor 714 4ch concat）+ mask extraction（tensor 727 1ch sigmoid）
- Native-res warp+blend shader：sample flow+mask at infer dim with bilinear in shader, warp prev/curr at native res

**Quality**：`verify_dump_interp` score **0.95**（≈ perfect midpoint）vs block-match **0%** effective interp。

**Latency**（RTX 3060 Laptop）：
- 256×128 推論 ~14ms（60fps DUAL OK）
- 128×128 推論 ~10ms（75fps server / 150fps display）
- 384×192 推論 ~25ms（RTX 4070+）
- 512×256 推論 ~40ms（高階卡）

**Stability**：β.6 `vkDeviceWaitIdle` 在 `drainOverlayStash` destroy 舊 VkImage 前修補了 30-60s `VK_ERROR_DEVICE_LOST`（overlay-resize use-after-free，Nsight Graphics `nv-aftermath-format` symbolize 出 `drawOverlayInRenderPass` 是真正 sample 的 site，不是原本誤判的 `vkfruc.frag`）。連續 7m49s 跑無 crash 驗證。

### §J.3.e.Y Tensor Core path（opt-in default-OFF）

`cooperative_matrix` 16×16 GEMM Conv2D，perf +7%，precision -70× regression（已 documented）。要啟用 `VIPLE_RIFE_VK_COOPMAT=1`。

### §B-NVOF — NVIDIA Optical Flow Vulkan

VK_NV_optical_flow extension，async cross-queue（zero CPU `vkWaitSemaphores`），perfLevel + grid env vars，SFIXED5→Q1 converter。

### §B1/§B2 production FRUC

HW path dual-present 60→120 production-ready，TRIPLE 60→180 (3x interp) infrastructure（opt-in，需 180Hz panel）。

### Android §A'-port

luma census + hierarchical diamond + warp 50/50 fallback —— 跟 Windows 端演算法完全對齊。Android §B-DUMP 跨平台 dump format。FRUC settings UI i18n × 27 locales（Bulgarian / 中央庫德 / Czech / German / Greek / Spanish / Persian / French / Hungarian / Indonesian / Italian / Hebrew / Japanese / Korean / Norwegian / Dutch / Polish / Portuguese (BR/PT) / Romanian / Russian / Swedish / Turkish / Ukrainian / Vietnamese / 簡體中文 / 繁體中文）。Security: dev-machine device serial redact + dump privacy notice。

### §K.X — Operational

- **Auto Wake-on-LAN toggle**：toggle OFF 時 `PcMonitorThread` polling cadence 從 ~3s 退到 60s，避免 NIC pattern-match 把 sleeping host 喚醒。右鍵「Wake PC」仍立即送 magic packet + 90s explicit-wake TTL fast cadence 讓 user 看到 host 上線。
- **Linux build alignment**：moonlight-qt Ubuntu noble + Qt 6 + g++ 13 完整 compile + link 通過。`runConv2DGpuTestStandalone` `_WIN32` guard、`nvofa` + `ncnn/stb` `INCLUDEPATH` `unix:!macx` 補上、`build-appimage.sh` 修 leftover `linuxdeployqt` sanity check + 補 `make qmake_all` 解 chicken-egg + 修 `make -C app -f Makefile.Release` 語法。
- **`scripts/` 整個 untrack**：dev/diagnostic helpers 不再進 public repo（`192.168.51.x` 私有 LAN IP 等 dev fingerprints 不外洩）。Build infra 搬遷到 `build-tools/`。

## Known issues

- §J.3.e.Y 4Y.6 Tensor Core path 預設 OFF（precision regression），要啟用 `VIPLE_RIFE_VK_COOPMAT=1`
- Path β TRIPLE 60→180 + Native RIFE 共存暫不支援（兩次 RIFE inference 時間 budget 不足，post-ship 處理）
- Linux 平台只在 WSL Ubuntu noble 驗測過；AMD GPU / Intel GPU / Linux native runtime 待後續 task
