## Fixes

- **§J.3.e.2.i.10e ROOT-CAUSE FIX** — `kFrucFramesInFlight` 4→2 revert（commit `6b9bb89`，v1.4.8）。v1.4.2 那個 2→4 bump 走錯方向，N=4 → 80 ms AVFrame hold cycle 直接對應使用者看到的「decodeMeanMs 穩定 80-100 ms」。改回 N=2 後 5-90 ms bimodal。
- **§J.3.e.2.i.10f Path D** — early AVFrame release via two-submit pattern（commit `9fd8c22`，v1.4.9，最終解）。renderFrame 拆 m_SlotCopyCmdBuf (~100 µs) + m_SlotCmdBuf (~20 ms) 兩段 graphics-queue submit，AVFrame hold cycle 從 40 ms → 0.2 ms，live test decodeMeanMs 0.47-30 ms typical 跟 D3D11 / FRUC OFF 同等級。
- **§J.3.e.X β.5.3 D-lite** — `inferDim` 從 DOWN-rounding 改 UP-rounding 到 /128 multiple（commit `9d4c237`，v1.4.1），解 RIFE-v4-lite Add_503 layer 在 inferDim 不是 /128 multiple 時 a.h round 不對齊撞 shape constraint 的問題。
- **§J.3.e.X β.11** — TRIPLE+ME `EDGE_AWARE_MV_THRESHOLD` 2.0 → 8.0（commit `fa46a88`），動態邊緣 8×8 馬賽克 fix；過濾 noise 維持 bilinear smooth，大 motion boundary (>2.83 px diff) 仍切 nearest pick 保 edge 銳利。
- **§N.4** — Server `/transfer/poll` non-blocking endpoint bug：`manager::poll(timeout=0)` 被誤判為「`<=0` 用 30 s default」而 block 30 s，client 15 s watchdog 永遠贏，polling 全 fail。改回真正 non-blocking 才解開整套。
- **§N.4b** — Server downloads dir 寫到 SYSTEM profile bug：服務 LocalSystem 帳號跑 `SHGetKnownFolderPath(nullptr)` 回 `C:\Windows\system32\config\systemprofile\Downloads`。改用 active console user token 拿正確 `C:\Users\<user>\Downloads`。
- **§N.5.bug (b)** — Server `sweep_stale_locked`：file transfer command 在 client 端 crash / 連線中斷時 60 s 後自動 release lock（避免下一個 client 來 transfer 卡死）。**(a)** Conscrypt SSL `TLSV1_ALERT_INTERNAL_ERROR` post-handshake 已用 client-side Supplier pattern 繞掉（每次 OkHttpClient 都新建避免 cached SSL state corruption）。
- **§M.1.f client-side** — moonlight-qt `session.cpp` 拆 `shouldQuit` 為 `shouldNotifyServerCancel`（graceful exit always send /cancel）+ `shouldQuitClientApp`（`quitAppAfter` pref）。Android leg `Game.java stopConnection()` 加同 pattern HTTP /cancel via `NvHTTP.quitApp()`。
- **§M.1.f server-side defensive race revert** — v1.4.39 revert `rtsp.cpp` `remove()` / `clear()` 的 `proc.clear_owner_uuid()` hook，因 race condition with normal stream setup（host log evidence 顯示 14:02:50 hook 在 14:02:47 launch 跟 14:02:50.197 RTSP session insert 之間 fire，把 just-set owner 抹掉）。`clear_owner_uuid()` public method 保留供未來 last_activity timeout 機制用。

## 主要技術里程碑

### §J.3.e.2.i.10 Path D — Vulkan + FRUC 80–100 ms regression root cause + 最終解

**症狀：** v1.4.0 之後使用者 live test 看到 RS_VULKAN + Native RIFE FRUC ON 時 decodeMeanMs 穩定卡 80-100 ms（D3D11 / FRUC OFF 時是 sub-ms），明顯 regression。

**Root cause (v1.4.8 commit `6b9bb89`):** v1.4.2 (`234d2aa`) 把 `kFrucFramesInFlight` 從 2 bump 到 4，以為展 CPU pipelining。但 Vulkan single-graphics-queue cmd buffer FIFO 序列執行，slot 多只展開 CPU pipelining、不增 GPU throughput。每 slot 跑完整 chain (~20 ms) 才 signal `vkf->sem[0]@V+1` 給 FFmpeg pool 釋放 image，所以 **AVFrame hold cycle = N × chain_time**。N=4 → 80 ms = 完全對應 user 看到的 100 ms。改回 N=2 後 hold cycle 40 ms，decodeMeanMs 5-90 ms bimodal、好的時段 5-15 ms。

**Final fix Path D (v1.4.9 commit `9fd8c22`):** 把 renderFrame 拆成兩段 graphics-queue submit：
- **Submit 1** (`m_SlotCopyCmdBuf`, ~100 µs)：只做 image→buffer copy + sem signal `vkf->sem[0]@V+1`，AVFrame 100 µs 內就被 release 給 FFmpeg pool。
- **Submit 2** (`m_SlotCmdBuf`, ~20 ms)：跑完整 RIFE chain + warp + present，不碰 vkf。

AVFrame hold cycle 從 40 ms → 0.2 ms。**Live test：decodeMeanMs 0.47-30 ms typical，sub-ms 多次出現，跟 v339 / D3D11 / FRUC OFF 同等級。** 能做到的關鍵是 §B-quality (d) 2026-05-06 已先把 DUAL+FRUC real-frame display 切到 `m_FrucCurrRgbBuf` via `m_RealCurrRgbDescSet`（不再吃 vkf->img[0] ycbcr sampler），所以 vkf->img[0] 只在 chain 開頭 ~100 µs 用一次。

### §J.3.e.X Path β quality + latency 收尾

- **β.5.2 Catmull-Rom bicubic 上採 flow + mask**（commit `a8b3204`，v1.4.1）— score 0.95 → 0.97-0.98，warp shader chain +0.4 ms 仍進 budget
- **β.5.3 D-lite UP-rounding inferDim** — RIFE-v4-lite Add_503 layer shape 對齊修正
- **β.9 TRIPLE 60→180 + Native RIFE coexist**（commit `99f86b5`，v1.4.6）— 拿掉 ctor 的 `m_RifeNativeMode + m_TripleMode` 互斥鎖；β.5.1 chain 重構成 `runOneInferAndWarp(t, outBuf, ds)` helper，TRIPLE 跑 2 次 RIFE inference（t=1/3 + t=2/3）
- **β.11 EDGE_AWARE_MV_THRESHOLD 2.0 → 8.0** — TRIPLE+ME 動態邊緣 8×8 馬賽克 fix

### §J.3.e.Y 4Y.7 C.1 Conv→BinaryOp(Mul) fusion

40 個 fuseable pair 由 Conv shader epilogue 吃掉，chain -2.6 ms（13.8 → 11.2 ms @ 256x128 baseline）。是 Tensor Core 路徑外 native RIFE 最有 ROI 的優化之一。

### §H.3 inferDim default 256 → 128 + UI honesty

mid-range GPU (RTX 3060 Laptop) 在 inferDim=256 chain ~20 ms > 16.7 ms (60fps slot)，dual-present 從 target 120 掉到 45.34 + p99 28 ms judder。inferDim=128 chain ~10 ms fit budget, fps 76.69。default 改 128，256 留給 RTX 4070+ opt-in。SettingsView.qml ComboBox text + RIFE checkbox ToolTip 一起對齊實測值。

### §M.1 Multi-user ownership guards + Web UI session dashboard

修「多人共用 server 時 A 客戶端意外殺到 B 的 Steam 遊戲」bug。`proc::proc._owner_uuid` per-launch lock，A 客戶端 /launch / /resume / /cancel 不能影響 B 正在玩的 session。Web UI 加 session dashboard：列當前 owner uuid + name + start time + force-disconnect 按鈕。

### §M.1.f Quit-key combo 退出後 server-side ownership 自動釋放

**Root cause:** moonlight-qt `session.cpp:1395-1397` 的 `shouldQuit` boolean 把「送 /cancel 給 server」跟「client app 退出」兩件事綁同條件 (`!m_UnexpectedTermination && m_Preferences->quitAppAfter`)，而 `quitAppAfter` pref 預設 false。所以 quit-key 退出時 server 從來不收到 /cancel，`proc::proc._owner_uuid` 永不清掉，下一台 paired device /launch 被 503 deny。

**Fix:**
- **moonlight-qt (v1.4.38):** 拆 `shouldQuit` 成 `shouldNotifyServerCancel`（`!m_UnexpectedTermination`，graceful exit always 送 /cancel）+ `shouldQuitClientApp`（`!m_UnexpectedTermination && quitAppAfter`，client app exit 與否維持原 pref）。
- **moonlight-android (v1.4.40):** `Game.java stopConnection()` 在 `conn.stop()` 之前先用 `xferHost / xferHttpsPort / xferServerCert` (file transfer cache 的 host auth) 建臨時 NvHTTP 呼叫 `quitApp()`，best-effort try-catch 包住。
- **server-side `clear_owner_uuid()` 方法** (v1.4.38) 保留供未來 last_activity timeout 機制用。
- **server-side rtsp.cpp 防禦 hook reverted** (v1.4.39) — 原本 `remove()` / `clear()` session_count==0 → `proc.clear_owner_uuid()` 因 race condition with normal stream setup 撞壞（hook 在 launch 跟 RTSP session insert 之間 fire 把 owner 抹掉）。

### §N In-stream 雙向檔案傳輸

完整 server + moonlight-qt + moonlight-android 三端：

- **Server**（v1.4.14 → v1.4.32，`Sunshine/src/file_transfer.{h,cpp}` + `nvhttp.cpp` + `confighttp.cpp` + `system_tray.cpp`）：per-paired-client command queue + token registry + chunked I/O + path sanitize + active-user Downloads dir resolver (WTSGetActiveConsoleSessionId + WTSQueryUserToken)。5 個 HTTPS endpoints (`/transfer/poll` / `/transfer/result` / `/transfer/blob` / `/transfer/progress`) + 4 confighttp endpoints (`/transfer` HTML / `/transfer/listing/latest` / `/transfer/pull` / `/transfer/progress-proxy`)。`system_tray.cpp` 加「Send file to client」/「Receive file from client」menu items。Windows native picker `src/platform/windows/fs_picker.cpp`（`IFileOpenDialog` COM wrapper）。Receive UI 升級成檔案總管樣式：path input + 麵包屑 + 資料夾可點進去 + folder「Pull as zip」自動 PowerShell `Compress-Archive` / Linux `zip`。
- **moonlight-qt client**（v1.4.15 → v1.4.33，`streaming/transfer/filetransferclient.{h,cpp}`）：2 s polling + LIST_DIR / DOWNLOAD_TO_CLIENT / UPLOAD_FROM_CLIENT / CANCEL dispatcher。**獨立 QThread + Qt event loop** — Session::exec() 跑 SDL main loop 整段 starve 主執行緒 Qt event loop，QTimer / QNAM 必須住自己的 thread。Stream-end teardown 先 clear QPointer 再 `abort()` 避免 same-thread synchronous `finished` signal callback null deref。SSL `onSslErrors` connect to pinned `m_Computer->serverCert` 比對通過就 `ignoreSslErrors`。每次 transfer 結束 `m_Nam->clearAccessCache()` 刷新連線池避免半關閉 socket。Folder zip 上傳 PowerShell `Compress-Archive` / Linux `zip` subprocess 自動清 temp。
- **moonlight-android client**（v1.4.19 起，`transfer/FileTransferClient.java`）：OkHttp polling + `MediaStore.Downloads` scoped-storage 讀寫 + streaming RequestBody。In-stream overlay（v1.4.38 §N UX polish）替代原 Toast，0-100% 進度全程可見、完成或失敗 3 秒後自動 hide。SSL state 對 Conscrypt corruption 用 Supplier pattern 每次拉 fresh wrapped OkHttpClient（v1.4.34 §N.5.bug client-side workaround）。

## Known issues

- **§N.5.linux mouse + overlay**：Linux client (Ubuntu VM AppImage) ↔ Windows host 場景，滑鼠 input forward 不傳（SDL2 mouse capture + xcb session 互動異常）、overlay 字元亂碼（AppImage 沒 bundle CJK font）。Keyboard input + file transfer endpoint 仍可用。留作 v1.4.41+ 處理。
- **§N.5.bug Scenario A**：Android Conscrypt `TLSV1_ALERT_INTERNAL_ERROR` alert 來源未抓到（已用 Supplier pattern 繞掉），underlying SimpleWeb / boost-asio handler entry/exit/try-catch diagnostic 仍 active。
- **§N.7 real zenity fs_picker**：v1.4.40 ship 只是 stub（回 nullopt），Linux server tray「Send to client」目前 noop。要 native Linux GUI session + zenity subprocess 整合，TODO。
- **§M.1.f.2 abnormal disconnect ownership timeout**：client crash / WiFi drop / power off 路徑因 client 永遠不會送 /cancel，目前 server `_owner_uuid` 仍會卡住。defensive RTSP hook v1.4.39 因 race 已 revert，後續用 `last_activity` timestamp + idle reconcile 機制（非 hook-on-teardown）補。
- **§J.3.e.X β.11**：threshold 8.0 是 hypothesis 1 baseline，待使用者實測決定 keep / 試 16（仍馬賽克）/ 退 4（edge 變糊）/ 改 push constant runtime tunable。其他 hypothesis (luma-gap backstop / big-motion bias / sub-pixel precision) 若需要再開。
- **Path β TRIPLE 60→180 + Native RIFE 共存效能** — β.9 Phase 1（互斥鎖解除 + 2× RIFE inference chain）已 ship，β.9 Phase 2（graph executor timestep-shared 前段 + timestep-dependent 後段 split 把 chain × 2 從 200% 降到 ~130%）仍 active，需要 chain @ 256x128 fit 60fps server 16.7 ms slot。
- **§J.3.e.Y 4Y.6 Tensor Core path** — 預設 OFF（precision -70× regression），要啟用 `VIPLE_RIFE_VK_COOPMAT=1`。
