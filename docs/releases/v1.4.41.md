# VipleStream v1.4.41 ‚Äî Idle reconcile + Linux fs_picker + Linux CJK fonts

**Release date**: 2026-05-14
**GitHub release**: <https://github.com/finaltwinsen/VipleStream/releases/tag/v1.4.41>
**Commit range**: `v1.4.40..v1.4.41`

A small follow-up release wrapping up the three Linux / cross-platform
follow-ups that were called out as known issues in v1.4.40.

## What's new

- **¬ßM.1.f.2 idle reconcile** ‚Äî server-side auto-release of ownership lock
  when the owner client goes silent (covers client crash / WiFi drop /
  power-off paths that never send `/cancel`)
- **¬ßN.7 real zenity `fs_picker` on Linux** ‚Äî Linux server tray "Send file
  to client" now actually opens a native file dialog instead of no-op
- **¬ßN.5.linux overlay CJK fonts in AppImage** ‚Äî bundles Noto Sans CJK
  inside the Linux AppImage so Traditional Chinese / Japanese / Korean
  text in overlays renders properly even on hosts without
  `fonts-noto-cjk` installed

## Fixes

- **¬ßM.1.f.2** ‚Äî covers the abnormal-disconnect path that ¬ßM.1.f
  (v1.4.38 PC client + v1.4.40 Android leg) couldn't reach. When a
  paired-device's `/launch` sets the owner lock and the client then
  disappears without graceful `/cancel`, the new idle watchdog
  auto-releases the lock after 60 s of "no RTSP session + no owner HTTP
  activity". Implementation lives in `Sunshine/src/process.{h,cpp}` ‚Äî
  pattern mirrors the existing Steam-game-exit watchdog (generation
  atomic for cancel-without-join). HTTP request handlers in
  `nvhttp.cpp` call a new `proc::proc.touch_activity(caller_uuid)` at
  the SSL-cert-UUID extraction sites for `/launch` and `/resume`; the
  watchdog also self-refreshes activity while
  `rtsp_stream::session_count() > 0` so long gameplay sessions with
  HTTP-quiet stretches don't false-trip the timeout.
- **¬ßN.7** ‚Äî replaces the v1.4.40 stub that returned `nullopt`
  unconditionally just to make the Linux server build link.
  `Sunshine/src/platform/linux/fs_picker.cpp` now spawns `zenity
  --file-selection` as a subprocess, with `--title` and `--filename`
  args from `open_options`, and parses the selected path back from
  stdout. Falls back to `nullopt` with a clear "install zenity" log if
  the binary isn't in PATH. Since Linux Sunshine runs as a per-user
  systemd service, `DISPLAY` / `WAYLAND_DISPLAY` /
  `DBUS_SESSION_BUS_ADDRESS` / `XDG_RUNTIME_DIR` are inherited
  automatically ‚Äî no extra session-handoff plumbing needed.
- **¬ßN.5.linux overlay CJK (partial)** ‚Äî `moonlight-qt/app/main.cpp`
  adds a Linux-only `QFontDatabase::addApplicationFont()` call (after
  `QGuiApplication` construction per the v1.2.27 ordering rule) that
  loads `NotoSansCJK-Regular.ttc` from a small candidate path list:
  AppImage's `$APPDIR/usr/share/fonts/opentype/` first, then standard
  system paths. Silent skip if not found.
  `moonlight-qt/scripts/build-appimage.sh` has cp logic that bundles
  the font from the builder's `/usr/share/fonts/opentype/noto/`, but
  **this release's builder did not have `fonts-noto-cjk` installed**,
  so the v1.4.41 AppImage falls back to the user's system fonts. CJK
  text renders correctly on Linux desktops that have `fonts-noto-cjk`
  installed system-wide; on minimal installs without it the overlays
  still show boxes. Builders running `apt install fonts-noto-cjk`
  before invoking `build-appimage.sh` get a self-contained AppImage on
  the next build (planned for v1.4.42+).

## Known issues / out-of-scope

- **¬ßN.5.linux mouse forward** ‚Äî Linux client mouse-event forwarding
  still doesn't work inside a Hyper-V VM console. Code analysis in
  v1.4.41 produced a hypothesis ranking (most likely:
  `SDL_SetRelativeMouseMode` silently falls back to fake-capture mode,
  which zeros out `xrel/yrel` deltas ‚Üí `LiSendMouseMoveEvent(0,0)`).
  Visual verification requires Linux GUI VM testing which is hard to
  automate, so the fix is deferred to v1.4.42+. Diagnostic log
  patches + Linux-default absolute-mouse-mode are the planned next
  steps. See `docs/TODO.md` ¬ßN.5.linux entry for the full hypothesis
  ranking + fix roadmap.
- **¬ßN.5.bug Scenario A** ‚Äî Android Conscrypt `TLSV1_ALERT_INTERNAL_ERROR`
  source unchanged (still worked around client-side via the v1.4.34
  Supplier pattern). Server-side BOOST_LOG entry/exit/try-catch
  diagnostic still pending.

## Upgrade

- Existing users on v1.4.40: no setting changes needed; the ¬ßM.1.f.2 idle
  watchdog is transparent. For Linux .deb installers, ensure `zenity`
  package is installed (`apt install zenity`) for the new "Send file to
  client" tray action.
- AppImage builders: `apt install fonts-noto-cjk` on the builder
  machine to bundle the CJK font (otherwise the AppImage falls back to
  whatever the user's host has installed).
